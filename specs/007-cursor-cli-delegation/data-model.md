# Data Model: Cursor CLI Agent Delegation

## Entities

### CursorSession

Represents one invocation of the Cursor CLI agent.

| Field           | Type                | Description                                                            |
| --------------- | ------------------- | ---------------------------------------------------------------------- |
| id              | string              | UUID generated by CloseClaw                                            |
| cursorChatId    | string or undefined | Cursor's own chat ID (from `agent ls`), populated after session starts |
| tmuxSessionName | string              | Name of the tmux session (for safe mode)                               |
| projectDir      | string              | Absolute path to the project directory                                 |
| prompt          | string              | The task description sent to Cursor                                    |
| mode            | "safe" or "trust"   | Execution mode                                                         |
| status          | SessionStatus       | Current state of the session                                           |
| startedAt       | Date                | When the session was spawned                                           |
| completedAt     | Date or undefined   | When the session finished                                              |
| timeoutMs       | number              | Configured timeout for this session                                    |
| platform        | BotPlatform         | Which platform the user is on                                          |
| senderId        | string              | The user who initiated the task                                        |
| outputLog       | string[]            | Captured output lines for final summary                                |

### SessionStatus

Enum: `spawning` | `running` | `waiting_approval` | `completed` | `failed` | `cancelled` | `timed_out`

### PermissionRequest

Represents a detected approval prompt.

| Field       | Type                            | Description                            |
| ----------- | ------------------------------- | -------------------------------------- |
| sessionId   | string                          | References the CursorSession           |
| promptText  | string                          | The permission prompt text from Cursor |
| detectedAt  | Date                            | When the prompt was detected           |
| respondedAt | Date or undefined               | When the user responded                |
| decision    | "accept" or "deny" or undefined | User's decision                        |

### SessionRecord

Persisted to `~/.closeclaw/cursor-sessions.json` for resume support.

| Field        | Type          | Description               |
| ------------ | ------------- | ------------------------- |
| id           | string        | CloseClaw session UUID    |
| cursorChatId | string        | Cursor's chat ID          |
| projectDir   | string        | Project directory         |
| prompt       | string        | Original task description |
| status       | SessionStatus | Final status              |
| createdAt    | string (ISO)  | Timestamp                 |

## State Transitions

```text
spawning → running → completed
                   → failed
                   → timed_out
                   → cancelled
         → waiting_approval → running (user accepts)
                            → running (user denies, agent skips)
                            → timed_out (approval timeout)
```

## Storage

- **Active sessions**: In-memory `Map<string, CursorSession>` keyed by `platform:senderId`
- **Recent sessions (for resume)**: Temp file at `os.tmpdir()/closeclaw-cursor-sessions.json` — written on session start, pruned of entries older than 24h on gateway startup. Not permanent storage; Cursor CLI manages its own session persistence natively.
- **Config**: `~/.closeclaw/config.json` field `cursorAgent.timeoutMs`
